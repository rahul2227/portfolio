## Decisions & Rationale
| Decision | Alternatives Considered | Why Chosen |
|----------|------------------------|------------|
| Legacy ONNX exporter (`dynamo=False`) | Dynamo-based exporter (torch 2.10 default) | Dynamo fails with LSTM dynamic shapes — known issue with bidirectional LSTMs |
| INT8 quantization | Full-precision ONNX (46 MB) | 4x smaller (11.6 MB), fits Pi RAM budget, acceptable accuracy for demo |
| Pad to max_width=2048 | Dynamic width (no padding) | Matches original training pipeline; model expects fixed-width tensors |
| `dependency-groups` over `project.optional-dependencies` | Standard PEP 621 optional-deps | uv uses `--group` flag and requires `dependency-groups` table |
| Synthetic sample images | Real IAM dataset images | IAM dataset not available locally; generated with Noteworthy font as placeholder |
| `_run_inference` writes to `st.session_state` | Direct render after inference | Results persist across tab switches without re-running inference |

## Learnings
- Torch 2.10+ requires `onnxscript` package even when using the legacy TorchScript exporter — it's imported at module level in `torch.onnx`
- `np.exp()` on random (non-log-softmax) values produces values > 1.0 — CTC decode confidence tests must use proper log-softmax inputs
- Preprocess pads all images to max_width, so tensor.shape[3] is always 2048 — tests for aspect ratio should check the intermediate step image, not the final tensor
- The `assert` in CNNToRNN.forward breaks ONNX tracing — must be removed for export
- ONNX validation across multiple widths (512, 1024, 2048) is essential for confirming dynamic axes work correctly

## Mistakes & Corrections
- **Mistake**: pyproject.toml used `[project.optional-dependencies]` instead of `[dependency-groups]`
  - **Fix**: Changed to `[dependency-groups]` which is what uv expects
  - **Prevention**: MEMORY.md already documents this — agents should reference it. Consider adding a pre-commit check.
- **Mistake**: ONNX export failed because torch 2.10 defaults to dynamo exporter
  - **Fix**: Added `dynamo=False` to `torch.onnx.export()` call
  - **Prevention**: MEMORY.md already documents this. The export script should have included it from the start.
- **Mistake**: `onnxscript` not in dependencies, causing import error even with `dynamo=False`
  - **Fix**: Added `onnxscript>=0.6.2` to export dependency group
  - **Prevention**: MEMORY.md documents this. Include in all torch ONNX export dependency lists.
- **Mistake**: Two tests failed — aspect ratio test checked padded tensor width (always 2048), confidence test used random non-log-softmax values
  - **Fix**: Aspect ratio test now checks step_images[1] (pre-padding). Confidence test now applies log-softmax to random values first.
  - **Prevention**: Be more careful about test assumptions matching implementation behavior.
